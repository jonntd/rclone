🔥 你的VPS专用115网盘配置（500MB内存+5GB硬盘）

===============================================
直接复制使用的命令：
===============================================

🎯 基础上传命令（推荐）：
rclone copy "你的文件" 115:目标文件夹/ --buffer-size=4M --max-buffer-memory=32M --transfers=1 --checkers=1 -v

🎯 后台上传命令（VPS推荐）：
nohup rclone copy "你的文件" 115:目标文件夹/ --buffer-size=4M --max-buffer-memory=32M --transfers=1 --checkers=1 --log-file=upload.log -v > upload_output.log 2>&1 &

🎯 同步文件夹命令：
rclone sync "本地文件夹" 115:云端文件夹/ --buffer-size=4M --max-buffer-memory=32M --transfers=1 --checkers=1 -v

===============================================
参数解释：
===============================================

--buffer-size=4M          每次读取4MB数据（很小，省内存）
--max-buffer-memory=32M    最多用32MB内存（你500MB内存的6%）
--transfers=1              一次只传1个文件（避免内存不够）
--checkers=1               只用1个线程检查文件（省CPU和内存）
-v                         显示详细进度

===============================================
❌ 这些参数对115网盘无效（不要用）：
===============================================

--multi-thread-cutoff=200M     # 115不支持多线程上传
--multi-thread-streams=4        # 115不支持多线程上传

原因：115网盘使用单线程分片上传，不支持rclone的多线程功能

===============================================
VPS实用技巧：
===============================================

📊 监控内存使用：
free -h

📊 监控上传进度：
tail -f upload.log

📊 查看后台任务：
jobs
ps aux | grep rclone

📊 停止后台上传：
pkill rclone

📊 创建临时swap（内存不够时）：
sudo fallocate -l 512M /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

===============================================
配置文件方式（可选）：
===============================================

创建 ~/.config/rclone/rclone.conf：

[115]
type = 115
# ... 你的115网盘登录信息 ...

# VPS超低配优化
buffer_size = 4M
max_buffer_memory = 32M
transfers = 1
checkers = 1

===============================================
常见问题解决：
===============================================

❓ 内存不够？
→ 减少buffer-size到2M：--buffer-size=2M --max-buffer-memory=16M

❓ 上传很慢？
→ 检查网络：ping 115.com
→ 增加重试：--retries=5

❓ 连接超时？
→ 增加超时时间：--timeout=10m --contimeout=60s

❓ VPS硬盘满了？
→ 直接云到云传输：rclone copy remote1:/ 115:/ --buffer-size=4M --max-buffer-memory=32M
